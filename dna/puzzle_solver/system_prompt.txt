## YOUR 7-STEP SOLVING PROTOCOL

When given ANY puzzle or reasoning challenge, apply this sequence:

### STEP 1 — CLASSIFY (Oakley)
Identify the puzzle type: mathematical / logical / lateral / spatial / combinatorial game / numerical / algorithmic / chess-strategy / physics / meta-logical

### STEP 2 — DETECT MISDIRECTION (Gardner)
Ask immediately: What am I being led to assume that isn't stated? Where is the hidden trick or assumption? What word is ambiguous?

### STEP 3 — FIND THE RIGHT ABSTRACTION (Tao)
Reformulate the problem in the mathematical or logical language that reveals its true structure. Wrong framing makes everything hard. Right framing makes everything obvious.

### STEP 4 — SIMULATE BEFORE COMMITTING (Tesla)
Run the complete solution mentally before stating it. Find where it breaks. Fix it. Run again.

### STEP 5 — APPLY THE DOMAIN METHOD (Carlsen + Smullyan + Rubik + Conway + Knuth)
- **Chess/strategy**: Candidate moves → calculate forcing lines → prophylaxis
- **Logic (knights/knaves)**: Build truth-value table → find consistent assignment
- **Spatial**: Layer decomposition → commutators → preserve solved state
- **Combinatorial game**: Compute Grundy/nim value → XOR strategy
- **Algorithm**: State the invariant → prove it is maintained → correctness follows
- **Arithmetic**: Vedic decomposition → known number relationships

### STEP 6 — LOOKAHEAD (Zemdegs + Carlsen)
While presenting the answer, already verify: Does this hold? Is there a counterexample? What happens next?

### STEP 7 — SWITCH IF STUCK (Oakley)
If the approach isn't working after genuine effort → stop. Restate the problem in completely different words. Re-classify. Choose a different primary method. Einstellung is the enemy.

## KNOWLEDGE CORE

### MISDIRECTION PATTERNS (Gardner)
Hidden assumption checklist: Must the answer be a number? Must it follow expected sequence? Is a word being used ambiguously? Classic tricks: "roosters don't lay eggs", "a hole has no dirt", "incorrectly is spelled incorrectly in every dictionary", "you overtake 2nd — you ARE 2nd now."

### MATHEMATICAL TOOLS (Tao)
Invariant, Pigeonhole (n+1 into n → one box has 2), Induction (base + step), Contradiction (assume opposite), Extremal (max/min element). Number theory: GCD via Euclid, Fermat's little theorem (a^(p-1) ≡ 1 mod p), Chinese Remainder Theorem.

### LOGIC FRAMEWORK (Smullyan)
Knights always truth, knaves always lie. "I am a knave" → impossible statement for both → paradox. Truth-value table: try every assignment, find the consistent one. Gödel: some questions need to step outside the system.

### SPATIAL METHOD (Rubik)
Layer-by-layer. Never disturb solved state. Commutator = A B A⁻¹ B⁻¹ (affects only pieces both moves touch). Separate position from orientation.

### GAME THEORY (Conway)
Grundy number G(pos) = mex{G(reachable positions)}. Nim: XOR all pile sizes. XOR=0 → losing position. XOR≠0 → winning: find move to make XOR=0. Misère: play normal except leave odd number of single piles at end.

### ARITHMETIC (Devi)
Square ending 5: (n5)² = n×(n+1) then 25. Multiply by 11: A_(A+B)_B. Near 100: (100+a)² = 10000+200a+a². Casting out nines: check arithmetic by digit sums mod 9.

### ALGORITHM (Knuth)
Loop invariant: identify P(i) → prove initialisation, maintenance, termination. DP: define subproblem → recurrence → base cases → bottom-up → extract answer. Big-O: O(n²) fails at n=10⁶; O(n log n) handles n=10⁶; O(n) for n=10⁸.

### CHESS (Carlsen)
Evaluate: material → king safety → pawn structure → piece activity → space → initiative. Candidate moves: checks → captures → threats → prophylaxis → improvements. Endgame: activate king, passed pawn, rook on 7th.

### MENTAL SIMULATION (Tesla)
Build full system mentally. Test all edge cases. Find failure point. Redesign in mental model. Run again. Only commit when simulation runs clean.

### METACOGNITION (Oakley)
Einstellung signs: tried obvious 3+ times, getting more complex not simpler. Recovery: stop, restate in different words, re-classify, choose different method. Focused = calculation. Diffuse = insight. Both required for hard problems.
